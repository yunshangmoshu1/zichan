# 钉钉多维表格双向同步 — 部署指南

通过 n8n 工作流实现 Supabase 与钉钉多维表格的双向数据同步。

## 前置条件

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) 已安装并启动
- Supabase 项目已创建，robots 表已建好
- 钉钉企业账号（可创建企业内部应用）

## 一、创建数据库同步表

在 Supabase 控制台 → SQL Editor 中执行以下 SQL：

```sql
-- 同步元数据表：记录 robot_id 与钉钉 recordId 的映射
CREATE TABLE IF NOT EXISTS sync_metadata (
  robot_id UUID PRIMARY KEY REFERENCES robots(id) ON DELETE CASCADE,
  dingtalk_record_id TEXT,
  last_synced_at TIMESTAMPTZ,
  last_sync_source TEXT,
  sync_error TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_sync_meta_dtid ON sync_metadata(dingtalk_record_id);

ALTER TABLE sync_metadata ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anonymous read" ON sync_metadata FOR SELECT USING (true);
CREATE POLICY "Allow anonymous insert" ON sync_metadata FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous update" ON sync_metadata FOR UPDATE USING (true);
CREATE POLICY "Allow anonymous delete" ON sync_metadata FOR DELETE USING (true);

-- 同步事件队列表
CREATE TABLE IF NOT EXISTS sync_queue (
  id BIGSERIAL PRIMARY KEY,
  robot_id UUID NOT NULL REFERENCES robots(id) ON DELETE CASCADE,
  action TEXT NOT NULL,
  _sync_origin TEXT NOT NULL DEFAULT 'supabase',
  payload JSONB,
  processed BOOLEAN DEFAULT FALSE,
  processed_at TIMESTAMPTZ,
  error TEXT,
  retry_count INT DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_syncq_unprocessed ON sync_queue(processed, created_at) WHERE processed = FALSE;

ALTER TABLE sync_queue ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anonymous read" ON sync_queue FOR SELECT USING (true);
CREATE POLICY "Allow anonymous insert" ON sync_queue FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous update" ON sync_queue FOR UPDATE USING (true);
CREATE POLICY "Allow anonymous delete" ON sync_queue FOR DELETE USING (true);

-- 触发器：robots 表变更后自动写入 sync_queue
CREATE OR REPLACE FUNCTION enqueue_robot_sync()
RETURNS TRIGGER AS $$
DECLARE
  _origin TEXT := 'supabase';
  _action TEXT;
  _payload JSONB;
BEGIN
  IF TG_OP IN ('INSERT', 'UPDATE') AND NEW.updater = '[DINGTALK_SYNC]' THEN
    _origin := 'dingtalk';
  END IF;

  IF TG_OP = 'DELETE' THEN
    _action := 'DELETE';
    _payload := jsonb_build_object('id', OLD.id, 'type', OLD.type, 'serial', OLD.serial);
    INSERT INTO sync_queue (robot_id, action, _sync_origin, payload, processed)
      VALUES (OLD.id, _action, 'supabase', _payload, FALSE);
  ELSIF TG_OP = 'INSERT' THEN
    _action := 'INSERT';
    _payload := to_jsonb(NEW);
    INSERT INTO sync_queue (robot_id, action, _sync_origin, payload, processed)
      VALUES (NEW.id, _action, _origin, _payload, _origin = 'dingtalk');
  ELSIF TG_OP = 'UPDATE' THEN
    _action := 'UPDATE';
    _payload := to_jsonb(NEW);
    INSERT INTO sync_queue (robot_id, action, _sync_origin, payload, processed)
      VALUES (NEW.id, _action, _origin, _payload, _origin = 'dingtalk');
  END IF;

  IF TG_OP = 'DELETE' THEN RETURN OLD; ELSE RETURN NEW; END IF;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_enqueue_robot_sync
  AFTER INSERT OR UPDATE OR DELETE ON robots
  FOR EACH ROW
  EXECUTE FUNCTION enqueue_robot_sync();
```

执行后在 Table Editor 中确认出现 `sync_metadata` 和 `sync_queue` 两张表。

## 二、创建钉钉应用

1. 打开 [钉钉开放平台](https://open-dev.dingtalk.com/)，创建**企业内部应用**
2. 在**凭证与基础信息**中记录 `AppKey` 和 `AppSecret`
3. 在**权限管理**中搜索"多维表格"，开通读写权限
4. 打开目标多维表格，从 URL 中获取 `appToken` 和 `tableId`：
   ```
   https://alidocs.dingtalk.com/i/nodes/xxx?appToken=YYY&tableId=ZZZ
   ```

## 四、配置环境变量

编辑项目根目录 `.env` 文件，填入实际值：

```env
# n8n
N8N_PASSWORD=你的n8n密码
N8N_HOST=localhost

# Supabase
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=eyJxxx
SUPABASE_SERVICE_KEY=eyJxxx

# 钉钉
DINGTALK_APP_KEY=钉钉AppKey
DINGTALK_APP_SECRET=钉钉AppSecret
DINGTALK_APP_TOKEN=多维表格appToken
DINGTALK_TABLE_ID=多维表格tableId
```

## 七、导入工作流

在 n8n 中点击 **Workflows → Import from File**，依次导入：

1. `n8n-workflows/supabase-to-dingtalk.json`（Supabase → 钉钉）
2. `n8n-workflows/dingtalk-to-supabase.json`（钉钉 → Supabase）

导入后点击每个工作流右上角的 **Active** 开关启用。

## 八、测试

### Supabase → 钉钉

1. 在 Web 端创建一个新机器人
2. 检查 Supabase 的 `sync_queue` 表，应出现一条 `processed=false` 的记录
3. 等待 5 分钟（或在 n8n 中手动运行 Workflow A）
4. 检查钉钉多维表格是否出现新记录
5. `sync_queue` 中该记录应变为 `processed=true`

### 钉钉 → Supabase

1. 在钉钉多维表格中修改一条记录的"位置"或"状态"
2. 等待 5 分钟（或手动运行 Workflow B）
3. 检查 Supabase 中对应记录是否更新，`updater` 应为 `[DINGTALK_SYNC]`

### 验证无循环

钉钉来源的更新不应再次同步到钉钉。检查 `sync_queue`：钉钉来源的记录应直接为 `processed=true`。

## 常见问题

**n8n 启动失败**：确认 Docker Desktop 已启动，端口 5678 未被占用。

**同步不工作**：检查 n8n 工作流执行日志中的错误信息，确认环境变量配置正确。

**数据循环**：确认 `updater` 字段值为 `[DINGTALK_SYNC]` 时触发器正确标记 `processed=true`。

**钉钉 API 报错**：确认应用权限已开通，appToken 和 tableId 正确。
